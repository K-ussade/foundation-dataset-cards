# Maintenance Standard

## Purpose

This standard defines update, correction, takedown, and deprecation expectations for public Foundation datasets.

## Required Maintenance Fields

| Field | Required Detail |
| --- | --- |
| Update cadence | Planned, one-time, periodic, event-driven, or deprecated. |
| Source refresh | How public source changes are detected or reviewed. |
| Correction path | How public errors are reported and corrected. |
| Takedown path | How privacy, source, or safety issues are escalated. |
| Deprecation conditions | When the dataset should be archived or replaced. |
| Maintainer role | Public role or authority label. |
| Last review date | Public date or planned status. |

## Takedown Triggers

Trigger review when:

- PII is suspected;
- private data is suspected;
- source terms change;
- public source becomes stale or inaccurate;
- sensitive infrastructure or operational detail appears;
- dataset is used for unsupported reliance;
- governance or safety review changes status.
