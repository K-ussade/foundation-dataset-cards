# Provenance Standard

## Purpose

This standard defines how public dataset cards document source, collection, licensing, and provenance without exposing private data or sensitive source detail.

## Required Provenance Fields

| Field | Required Detail |
| --- | --- |
| Source category | Public document, public civic reference, public report, synthetic example, approved sanitized sample, or other approved class. |
| Source name | Public source name or approved generic label. |
| Source URL | Public URL when available and safe. |
| Collection date | Date or date range when public. |
| Collection method | Public-safe method description. |
| License or terms | License posture, source terms, or review status. |
| Transformation | Public-safe sanitation, aggregation, labeling, or formatting summary. |
| Exclusions | Private, sensitive, sealed, or unreviewed material excluded. |
| Review status | Source, privacy, licensing, and governance review status. |

## Forbidden Provenance Detail

Do not publish:

- private source locations;
- private partner records;
- raw private submissions;
- exact sensitive infrastructure locations;
- security-sensitive operational details;
- secret collection credentials;
- private review notes;
- sealed method details.
