# Dataset Card Standard

## Purpose

This standard defines required content for Foundation-aligned Hugging Face dataset-card templates and future reviewed dataset cards.

## Required Sections

Every dataset-card template must include:

| Section | Requirement |
| --- | --- |
| Status | Planned, scaffolded, experimental, released, deprecated, or private/not-public. |
| Dataset summary | Public-safe summary without release overclaiming. |
| Data source | Public or reviewed source basis. |
| Collection process | How data was collected or why the field is not applicable. |
| Allowed uses | Public-safe uses after release. |
| Forbidden uses | Prohibited, unsupported, or high-risk uses. |
| Privacy exclusions | Explicit exclusions for private and sensitive data. |
| PII statement | PII presence, absence, review status, or not applicable. |
| Bias/limitations | Known source, coverage, representation, and use limitations. |
| Governance review status | Human review status. |
| Maintenance plan | Update, correction, takedown, and deprecation plan. |
| GitHub companion link | Link placeholder for companion docs. |
| `218.network` placeholder | Context link placeholder or planned status. |

## Claim Rules

- Do not claim a dataset exists unless the public Hugging Face dataset repository exists.
- Do not publish dataset files in this repository.
- Do not include donor, student, school, volunteer, customer, private operations, telemetry, or sealed IP material.
- Planned names must stay marked planned.
