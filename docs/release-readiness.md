# Release Readiness

## Purpose

This standard defines the public documentation gates required before a dataset can be described as released.

## Required Gates

| Gate | Required outcome |
| --- | --- |
| Dataset card | Complete card with source, collection process, privacy, limitations, governance status, and maintenance. |
| Sanitation review | Privacy and sensitive-data exclusions are reviewed. |
| Governance review | Human Foundation data release authority approves public status. |
| Release notes | Public release notes exist and avoid overclaims. |
| GitHub companion | Public companion README and checklist are linked. |
| Hugging Face artifact | Public dataset repository exists before `released` status is used. |

## Non-Claim Rule

A dataset-card template or planned name is not a dataset release.
