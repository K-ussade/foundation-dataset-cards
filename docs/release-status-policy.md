# Release Status Policy

## Purpose

This policy standardizes how dataset status is described in dataset cards, release notes, profile documentation, and transparency reports.

## Statuses

| Status | Meaning |
| --- | --- |
| Planned | Name, purpose, and review path are documented; no public dataset artifact is claimed. |
| Scaffolded | Public card or companion documentation exists; no dataset release is claimed unless stated. |
| Experimental | Limited reviewed dataset artifact exists for evaluation with explicit limitations. |
| Released | Public dataset artifact has approved card, privacy review, maintenance plan, companion links, and governance review status. |
| Deprecated | Artifact remains visible only with replacement, sunset, or archival context. |

## Required Status Handling

- Planned names must not be described as collected, sanitized, uploaded, downloadable, maintained, or released.
- Scaffolded cards must state that no dataset artifact is released unless a public Hugging Face dataset exists.
- Experimental datasets require limitation, privacy, and monitoring language.
- Released datasets require complete public release documentation.
- Deprecated datasets require replacement, sunset, or archival context.
