# Sanitation Standard

## Purpose

This standard defines sanitation requirements before any dataset or sample can be considered for public release.

## Required Checks

| Check | Required outcome |
| --- | --- |
| Source review | Source is public, licensed, synthetic, or explicitly approved. |
| PII review | Personal and sensitive data are absent or handled under approved public release rules. |
| Sensitive location review | Exact sensitive infrastructure locations are excluded. |
| NEURONA security review | Security-sensitive operational details are excluded. |
| Donor/student/volunteer review | Donor, student, school private, and volunteer private data are excluded. |
| Customer/sealed-IP review | Customer data and sealed YOSO-YAi LLC IP are excluded. |

## No-Sample Rule

Do not add sanitized samples to this repository. Use templates and synthetic examples only unless a separate human release decision explicitly authorizes a public sample in a proper release repository.
