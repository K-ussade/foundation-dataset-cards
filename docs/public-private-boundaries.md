# Public / Private Boundaries

## Purpose

This document defines what may and may not appear in public dataset-card templates, examples, and future reviewed cards.

## Public-Safe Material

- Template structure.
- Public source and provenance language.
- Sanitation and governance review checklists.
- Placeholder metadata.
- Synthetic examples clearly labeled synthetic/public-safe.
- Public GitHub and Hugging Face links after artifacts exist.

## Blocked Material

- Dataset files.
- Raw datasets.
- Private datasets.
- Unapproved sanitized samples.
- Donor private data.
- Student data.
- School private data.
- Volunteer private data.
- Customer data.
- Private Foundation operations.
- Private partner records.
- Private telemetry.
- Exact sensitive infrastructure locations.
- Security-sensitive NEURONA operational details.
- Private training corpora.
- Sealed YOSO-YAi LLC IP.
- Secrets, tokens, and API keys.

## Boundary Rule

Summaries, examples, screenshots, generated outputs, and samples inherit the source boundary until explicit human data release review clears them for public use.

Required exclusion language for dataset-card review: dataset files, raw data, private data, unapproved samples, donor private data, student data, volunteer private data, customer data, private telemetry, exact sensitive infrastructure locations, sealed YOSO-YAi LLC IP, secrets, and security-sensitive NEURONA operational details are not public-safe.
